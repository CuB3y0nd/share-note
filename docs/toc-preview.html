Designing a Better Share Note TOC

This preview page mirrors the current direction of the generated TOC. It includes a floating desktop rail, a small-screen drawer, and a top collapsible variant.

Scroll through the article and watch the active section update. The current item and its ancestor chain should keep the shadowed background while sibling branches collapse away.

Problem Shape

The original implementation pushed the article inward because the TOC lived inside the readable-width layout. That works for wide app shells, but it breaks note reading when the content column is already constrained.

The corrected layout moves the TOC outside the readable-width area and lets Obsidian's file width continue to define the article column.

Layout Goal

The rail should feel attached to the page without compressing the prose. On small screens it needs to switch to an alternate presentation instead of fighting for horizontal space.

Tradeoffs

There is still a balance between visibility and intrusion. The rail needs enough contrast to be discoverable, but not so much chrome that it overpowers the note itself.

Interaction Goal

Readers should know where they are at a glance. That means the active section must be obvious, and the parent chain needs to remain expanded while off-path branches collapse cleanly.

Visual Adjustments

This pass tightens the typography, aligns the dashed guide with the numbering axis, and hides the native scrollbar so the card reads as a cleaner editorial component.

Smaller Type

The TOC text now sits a little lower in scale than the previous version. That helps it read as navigation rather than as primary content.

Badge Axis Alignment

The dashed vertical guide is positioned off the badge centerline rather than from an arbitrary container border. That makes the numbering and guide feel mechanically aligned.

Nested Rhythm

Nested sections still need enough padding to breathe, but the guide should preserve a clear visual spine from parent into child groups.

Active Path Styling

The active node and all ancestors receive a shadowed background state. This gives the reading path a stronger sense of continuity as you move deeper into the document.

Mobile Modes

Small screens need deliberate alternatives. This preview lets you switch between hidden, drawer, and top collapsible modes from the toolbar.

Drawer Mode

The drawer button stays out of the reading flow and opens a panel from the lower edge. This is useful when you want the TOC available but not always visible.

Collapse Mode

The collapsible header keeps the TOC near the article top. It is simpler than the drawer and works well when a note benefits from a quick structural overview before reading.

Verification Notes

This page is only a static preview, so it is not wired into the plugin build. It exists to let you inspect the visual direction and the runtime TOC behavior without needing the full Obsidian share flow.

If the feel is right here, the same classes and runtime path styling can stay in the generated page output.
